RRAM Simulation (CrossSim)

RRAM Background

Resistive random-access memory (ReRAM or RRAM) is based on Memristor.

History of Memristor

1971, Leon O. Chua in UC Berkeley, predicted its existence IEEE, “Memristor-The missing circuit element”

Memristor is first introduced by Leon O, Chua in 1971, combining Memory and Resistor.

•2008, HP company’s laboratory made the first memristor Nature, “The missing memristor found”

Component Structure

•Two Metal layer

• Intermediate layer:

Not doped region: Pure TiO2, high resistance

Doped region: with oxygen deficiencies TiO2-X, low resistance

• Stored information: resistance

• Memristor crossbar : each node is a memristor

In-memory Computing

•Apply N voltage signals in the rows

•The crossbar is a N×M matrix. Each intersection node is a memristor

• According to Kirchhoff’s Current Law, The current of column :

: the conductance of memristor

Input vector: 1×N (voltage signals)

Vector-matrix multiplication: (1×N) × (N×M)

Output vector: 1×M (current signals)

Application in AI (Artificial Intelligence)

•Each memristor: a biological neuron In-memory computing

•Crossbars accelerate 3 operations:

1) Vector-matrix multiplication

2) Matrix-vector multiplication

3) Outer product of two vectors

• Building artificial intelligent algorithm with memristor:

1) Backpropagation

2) Liquid state machine

3) Restricted Boltzmann machines

Cross-Sim

CrossSim is a crossbar simulator designed to model resistive memory crossbars for both neuromorphic computing and digital memories. It provides a clean python API so that different algorithms can be built upon crossbars while modelling realistic device properties and variability.

Three operations:

Vector *Matrix: .run_xbar_vmm

Matrix*Vector: .run_xbar_mvm

Vector*Vector: .update_matrix
